# 이미지 n×m 분할 — 설계

날짜: 2026-07-23
대상: 웹앱(`index.html`) 전용. 데스크톱 셸(`shell/`)에는 추가하지 않는다.

## 목적

이미지 한 장을 사용자가 지정한 가로·세로 조각 수로 균등하게 잘라, 조각을 개별 또는
ZIP으로 한꺼번에 내려받는다. 자르면서 형식·품질을 지정해 용량도 함께 줄인다.

## 범위

포함:
- 자체 드롭존을 갖는 독립 탭 (드래그 · 클릭 · Ctrl+V)
- 가로/세로 조각 수 입력에 따라 원본 위에 실시간 갱신되는 가이드라인 오버레이
- [분할하기] 실행 → 조각별 썸네일·크기·용량 목록
- 조각 개별 저장 + 전체 ZIP 저장
- 출력 형식(자동/WebP/JPEG/PNG) + 품질 슬라이더

제외 (YAGNI):
- 가이드라인을 드래그해 불균등 분할 — 요청은 "조각 수에 따른 균등 분할"이다
- 조각 리사이즈 / 목표 용량(KB) 이진탐색 — 분할은 원본 픽셀 보존이 목적이고,
  조각마다 6회 이진탐색은 100조각에서 지나치게 느리다
- 데스크톱 셸 이식

## 배치

`index.html` 탭 목록 7번째 항목으로 `분할하기`를 추가한다. 순서는
압축하기 · 배경·요소 · **분할하기** · SVG 변환 · PDF 만들기 · Base64 도구 · 메타데이터.

- 버튼 `#tabSplit`, 패널 `#panel-split`, 해시 키 `split`
- `tabs` 배열에 `{ key: 'split', label: '이미지 분할', desc: '사진을 격자로 잘라 한꺼번에 저장해요' }` 추가
- Ctrl+V 붙여넣기 분기(현재 `!$('#panel-svg').hidden` 식으로 분기하는 곳)에 split 분기 추가

## 화면 구성

```
┌ 이미지 분할 ────────────────────────┐
│ [드롭존]                             │
│ ┌──────────┬──────────┬──────────┐  │  ← 가이드라인 오버레이
│ ├──────────┼──────────┼──────────┤  │
│ └──────────┴──────────┴──────────┘  │
│ 가로 [3]        세로 [2]             │
│ 프리셋: 2×2  3×3  1×3  3×1          │
│ 형식: 자동 WebP JPEG PNG            │
│ 품질: ────●──── 85%                 │
│           [ 분할하기 ]               │
├─ 결과 6조각 · 원본 1.2MB → 합계 480KB┤
│ [썸네일 1-1 · 640×480 · 82KB · 저장] │
│ ...  (n×m 격자 그대로 배치)          │
│ [ 전체 ZIP 저장 ]                    │
└──────────────────────────────────────┘
```

가이드라인은 어떤 사진 위에서도 보이도록 두 겹으로 긋는다 — 흰색 3px을 먼저 긋고 그 위에
강조색(청록) 1px을 겹친다. 밝은 사진에서는 청록선이, 어두운 사진에서는 흰선이 보인다.

결과 목록은 CSS Grid로 `grid-template-columns: repeat(cols, 1fr)`를 인라인 지정해
실제 분할 배치와 같은 모양으로 보여준다.

## 분할 계산

경계 좌표는 소수 누적이 아니라 반올림 분수로 잡는다.

```js
function edges(total, n) {            // n+1개 경계
  var out = [];
  for (var i = 0; i <= n; i++) out.push(Math.round(total * i / n));
  return out;
}
```

- 조각 (r, c)의 영역: `x = X[c]`, `y = Y[r]`, `w = X[c+1] - X[c]`, `h = Y[r+1] - Y[r]`
- 나누어떨어지지 않아도 조각 간 크기 차이는 1px 이내이고, **빈틈·겹침 없이 원본 전체를 덮는다**
- `Math.floor(w / cols)` 방식은 오른쪽·아래 끝 픽셀이 잘려 나가므로 쓰지 않는다
- 같은 `edges()`를 미리보기 오버레이와 실제 분할이 공유해, 보인 대로 잘린다

## 이미지 로딩

기존 `decodeToSource(file)`를 그대로 쓴다. EXIF 방향 보정(`imageOrientation: 'from-image'`)과
HEIC 등 미지원 형식 오류 메시지를 이미 처리한다. 투명도 판정은 기존 `detectAlpha(source)`.

## 인코딩

조각마다 정확한 크기의 캔버스를 만들고 `drawImage(source, sx, sy, sw, sh, 0, 0, sw, sh)`로
해당 영역만 그린 뒤 기존 `encodeCanvas(canvas, type, quality)`로 인코딩한다.

형식 `자동` 규칙은 기존 `resolveFormat`과 동일하게 맞춘다:
- 투명도 있음 → WebP (미지원 브라우저면 PNG)
- 투명도 없음 → WebP (미지원 브라우저면 JPEG)

JPEG로 인코딩할 때 원본에 투명도가 있으면 흰색으로 배경을 채운 뒤 그린다.
PNG는 품질 슬라이더의 영향을 받지 않으므로 PNG 선택 시 슬라이더를 비활성화한다.

## 다운로드

- 개별: 기존 `saveBlob(blob, name)`
- 전체: 기존 `makeZip([{name, blob}...])` → `saveBlob(zip, ...)`
- 조각 이름: `<원본이름>_<행>-<열>.<확장자>` (row-major, 1부터). 예: `사진_1-1.webp`
- ZIP 이름: `<원본이름>-분할.zip`

## 제한과 오류 처리

- 가로·세로 각각 1~10 (입력값은 clamp), 총 조각 100개 이하
- 1×1은 허용하되(원본 재인코딩) 프리셋에는 두지 않는다
- 인코딩 실패 조각은 건너뛰고 토스트로 실패 개수를 알린다. 나머지 조각은 정상 저장 가능
- 열 수 없는 파일은 `decodeToSource`의 오류 메시지를 토스트로 표시
- 새 이미지를 넣거나 조각 수를 바꾸면 이전 결과의 `URL.revokeObjectURL`을 호출하고 목록을 비운다
- 분할 실행 중에는 [분할하기]를 비활성화하고 진행 상태를 표시한다

## 검증

이 저장소에는 JS 테스트 하네스가 없다. Chrome DevTools MCP로 실제 브라우저에서 `index.html`을
열어 실측 확인한다.

1. 알려진 크기의 테스트 이미지(예: 1000×700, 3으로 나누어떨어지지 않음)를 넣는다
2. 3×2 분할 실행 후 조각 6개가 생기는지 확인
3. **조각 폭의 합 = 1000, 높이의 합 = 700** 확인 (빈틈·겹침 없음 검증)
4. 각 행의 폭 구성이 동일하고 각 열의 높이 구성이 동일한지 확인
5. 전체 ZIP 저장 후 항목 수 6 확인
6. 미리보기 가이드라인 위치가 `edges()` 결과와 일치하는지 확인
7. 형식 자동/WebP/JPEG/PNG 각각에서 조각 MIME 타입 확인
8. 다크 모드에서 가이드라인이 보이는지 육안 확인
