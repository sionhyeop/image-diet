압축 방식

이 브라우저는 WebP 저장을 지원하지 않아 JPEG로 대체돼요.

설정을 바꾸면 목록의 이미지가 자동으로 다시 압축돼요. 재인코딩된 결과에서는 EXIF(위치정보 등) 메타데이터가 제거돼요 — ‘원본 유지’로 표시된 항목은 원본 그대로예요.

    이미지 → SVG 벡터 변환

    색 면의 외곽선을 추적해 진짜 벡터 패스로 바꿔요. 로고·아이콘·일러스트처럼 색 수가 적은 이미지에 잘 맞고, 사진은 포스터화된 그림처럼 표현돼요.

    곡선 처리

    이미지 → Base64 (Data URI)

    파일을 압축 없이 그대로 문자열로 바꿔요. 압축까지 하려면 ‘압축하기’ 탭의 Base64 복사를 쓰세요.

    Base64 → 이미지

    Data URI 또는 순수 Base64 문자열을 붙여넣으세요. 이미지 형식은 자동으로 알아내요.

    🔒 모든 변환은 내 브라우저 안에서만 일어나요 — 어떤 이미지도 서버로 전송되지 않아요. 인터넷이 끊겨도 동작해요.

    GitHub Pages로 배포하기
    1. GitHub에서 새 저장소를 만드세요 (예: image-diet).
    2. 이 폴더에서 아래 명령을 실행하세요. 
      git init -b main
git add .
git commit -m "image diet"
git remote add origin https://github.com/<아이디>/image-diet.git
git push -u origin main
    3. 저장소 Settings → Pages에서 Source를 Deploy from a branch, Branch를 main / (root)로 지정하면 몇 분 뒤 https://<아이디>.github.io/image-diet/에서 열려요.

    index.html 파일 하나만 따로 저장해도 어디서든 오프라인으로 동작해요.
